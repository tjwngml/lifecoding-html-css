## Prob 1.

### HTML 이미지 경로 문제 원인 분석 ( `환경에 따른 루트(/)의 정의 차이` )

- 문제점 : 프로젝트 루트에 html 파일과 public/images/이미지 폴더가 위치하여 이미지를 불러 올 때 상대경로와 절대경로 둘 중 무엇을 쓰든 이미지가 잘 불러와 질 것이라고 예상하였으나 상대경로를 이용한 방식만 이미지가 정상적으로 불러와지는 문제가 발생

- 원인 핵심 요약 : 브라우저에서 파일을 직접 열면 ( 더블 클릭하면 ) 경로의 시작점인 루트 ( / ) 가 프로젝트 폴더가 아닌 컴퓨터의 최상위 드라이브 ( C : / ) 또는 ( / ) 가 된다.

- 이런 현상이 발생하는 이유 :
  - 파일 시스템 (`file : ///`) : 브라우저는 현재 열린 HTML 파일이 담긴 폴더가 전체 세상의 끝이라고 생각하지 않는다 .따라서 `/` 를 입력하면 “ 이 컴퓨터의 하드디스크의 시작점” 으로 점프해 버린다.
  - 웹 서버 (`http://`) : Live Server 같은 도구를 쓰면, 서버는 프로젝트 폴더를 하나의 독립된 ‘ 웹사이트 세상’ 으로 마든다. 이때부터 `/` 는 “이 웹사이트 (프로젝트) 의 최상단” 을 의미하게 된다.

- 해결책 :
  1. 로컬 작업 시 : 가급 적 상대경로를 사용하기. 그래야 서버환경이든 로컬 환경이든 잘 작동한다.
  2. 개발 환경 활용 : vscode 의 Live Server 확장 프로그램을 사용하여 항상 `http: [localhost](http://localhost)` 환경에서 작업하는 습관을 들이는 것이 좋다. ( 실제 배포 환경과 가장 유사하다 )
